# たまごっち機能 設計ドキュメント

## コンセプト

既存の yamagotchi ダッシュボード（ドット絵キャラ + エージェント状態表示）に、本物のたまごっちのような育成パラメータ・お世話機能・ミニゲームを追加する。まずは業務連動なしの純粋なたまごっちとして完成させ、後から業務データと接続する。

## パラメータ

5つのパラメータを管理する。各パラメータは 0〜100 の整数値。全パラメータは常に 0〜100 にクランプされる（下限0、上限100を超えない）。

| パラメータ | アイコン | 意味 | 自然減衰 | 回復手段 |
|-----------|---------|------|---------|---------|
| おなか | 🍙 | 空腹度 | -1/5分 | ごはんボタン |
| きげん | 😊 | 機嫌 | -1/10分 | ミニゲームで勝利 |
| うんち | 💩 | 排泄（高いほど汚い） | +1/15分 | おそうじボタン |
| ねむけ | 💤 | 眠気（高いほど眠い） | +1/20分 | 就寝で回復（後述） |
| 体調 | 🏥 | 健康状態 | 他パラメータが悪いと低下 | くすりボタン |

### 減衰ルール

- リアルタイム進行。ブラウザが閉じていてもlocalStorageの最終更新時刻から経過時間を計算して反映する
- 体調は他のパラメータから導出:
  - おなか < 20 → 体調 -1/5分
  - うんち > 80 → 体調 -1/5分
  - ねむけ > 80 → 体調 -1/10分
- 全パラメータが良好な状態（おなか >= 50, きげん >= 50, うんち <= 50, ねむけ <= 50, 体調 >= 50）が30分以上続くと、きげんが自然回復（+1/10分）。計測には `allGoodSince` タイムスタンプを使用する
- 就寝中はおなか・きげんの減衰速度が半減する（-1/10分、-1/20分）。うんち・ねむけの増加も半減する

### 就寝ルール

- 自動就寝: 22:00 に自動で `isSleeping=true` になる
- 自動起床: 7:00 に自動で `isSleeping=false` になる
- でんきボタン: 手動で就寝/起床を切り替える。手動操作後は次の自動イベント時刻まで手動状態を維持する（22:00に手動で起こしても、翌7:00に自動起床処理は走る）
- 就寝中はねむけが -1/5分 で回復する
- 就寝中もごはん・おそうじ・くすりボタンは使用可能（使うとキャラが一瞬起きるアニメーション演出だが、`isSleeping` 状態は変わらない）

### 表情マッピング

パラメータの組み合わせでキャラの表情が変わる。たまごっちパラメータ由来の表情は既存の `CharacterState` とは別に `TamagotchiExpression` 型として管理する。`expression.ts` がパラメータから `TamagotchiExpression` を返し、`getSprite()` は `CharacterState | TamagotchiExpression` どちらでも受け取れるようにする。

| 条件 | 表情 | TamagotchiExpression値 | sprites.ts対応 |
|------|------|----------------------|----------------|
| 全パラメータ良好 | にこにこ | "happy" | normal（既存）を流用 |
| おなか < 30 | もぐもぐ顔 | "hungry" | 新規追加 |
| きげん < 30 | しょんぼり | "sad" | 新規追加 |
| うんち > 70 | 困り顔 | "dirty" | 新規追加 |
| ねむけ > 70 | うとうと | "drowsy" | idle（既存）を流用 |
| 体調 < 30 | 具合悪い | "sick" | error（既存）を流用 |
| 就寝中 | Zzz | "sleeping" | idle（既存）を流用 |

優先度: 就寝中 > 体調 > ねむけ > おなか > うんち > きげん

## ボタン

画面下部にアイコンベースのボタンを配置する。

| ボタン | アイコン | 機能 |
|--------|---------|------|
| ごはん | 🍙 | おなかを +20 回復 |
| あそぶ | 🎮 | ミニゲーム画面へ遷移 |
| おそうじ | 🧹 | うんちを -30 |
| くすり | 💊 | 体調を +30 回復 |
| でんき | 💡 | 就寝ON/OFF切り替え |
| ステータス | 📊 | パラメータ一覧表示 |

### ボタン制約

- ごはん: 直近10分以内に3回使用したら使用不可（`feedCount` と `lastFedAt` で管理）。おなかが90以上のときも使用不可
- くすり: 体調が70以上のとき使用不可
- おそうじ: うんちが10以下のとき使用不可

## ミニゲーム

「あそぶ」ボタンから3つのミニゲームを選択できる。勝利できげん +15、敗北でも +5。

### じゃんけん

- グー✊ / チョキ✌️ / パー🖐️ を選択
- 3回勝負（引き分けはカウントしない、やり直し）。2勝で勝ち
- キャラがドット絵で手を出すアニメーション

### 方向当て

- キャラが上下左右のどれかを向く
- プレイヤーが同じ方向を当てる
- 5回中3回正解で勝ち

### もぐらたたき

- 3x3のグリッドにキャラが出現
- 出現間隔: 1.2秒ごとに1匹出現、表示時間は1.0秒（間に合わなければ消える）
- 同時に2匹まで表示される場合がある
- 10秒間で5匹以上タップで勝ち

## 永続化

localStorageに以下を保存する。

```
yamagotchi-state: {
  params: { hunger: number, mood: number, poop: number, sleepy: number, health: number },
  lastUpdated: ISO8601文字列,
  isSleeping: boolean,
  allGoodSince: ISO8601文字列 | null,
  feedCount: number,
  feedWindowStart: ISO8601文字列 | null（ごはん連続制限の10分ウィンドウ開始時刻）,
  lastManualSleepAt: ISO8601文字列 | null（手動でんき操作の時刻。次の自動境界まで手動状態を維持）,
  totalAge: number（分単位、起動からの累計。現フェーズではUI表示なし。将来の進化判定用に記録のみ）
}
```

- 保存タイミング: パラメータ変更時 + 1分ごとの定期保存
- ブラウザ再開時: lastUpdated から経過時間を計算し、その分だけ減衰を一括適用。`allGoodSince` も経過時間に応じて評価する
- 初回起動時のデフォルト値: おなか=80, きげん=80, うんち=0, ねむけ=0, 体調=100

## 型設計

### 既存型との共存

既存の `CharacterState`（"normal" | "processing" | "error" | "resolved" | "idle"）はエージェント状態表示用にそのまま維持する。たまごっちの表情は新しい `TamagotchiExpression` 型で管理する。

```
TamagotchiExpression = "happy" | "hungry" | "sad" | "dirty" | "drowsy" | "sick" | "sleeping"
```

`getSprite()` のシグネチャを `(state: CharacterState | TamagotchiExpression) => SpriteAnimation` に拡張する。既存の `CharacterState` 値はそのまま動作し、新しい `TamagotchiExpression` 値は対応するスプライトを返す（一部は既存スプライトを流用）。

`page.tsx` では、たまごっちモードが有効な場合は `TamagotchiExpression` を、エージェント状態表示モードの場合は `CharacterState` を `getSprite()` に渡す。
