# yamagotchi 設計ドキュメント

## コンセプト

Claudeエージェントの状態をたまごっち風ドット絵キャラで表示するブラウザダッシュボード。
卵型フレームの中に32x32カラードット絵キャラが1体いて、エージェントの状態に応じて表情やアニメーションが変わる。

## アーキテクチャ

```
[inta] APIエンドポイント ──(HTTP)──> [yamagotchi] ブラウザ表示
```

- inta 側に `/api/status` エンドポイントを追加
- yamagotchi 側から数秒間隔でポーリング
- 取得した状態に応じてドット絵キャラの見た目が変化

## エージェント状態

表示する情報:

1. **プロセス状態**: 起動中 / 停止中 / エラー
2. **タスク種類**: Slack対応中 / コーディング中 / 待機中 など

## キャラ状態パターン

| 状態 | 表情 | アニメーション |
|------|------|---------------|
| 待機中 | のんびり | ゆらゆら |
| Slack対応中 | しゃべってる | 口パク |
| コーディング中 | 集中 | タイピング |
| 停止中 | 寝てる | Zzz |
| エラー | 具合悪そう | ふらふら |

## 技術スタック

### フロントエンド (yamagotchi)

- **bun** ランタイム
- **Next.js** (shadcn プリセット使用)
- **shadcn/ui** (zinc テーマ, JetBrains Mono フォント)
- **Canvas API** でドット絵描画 (32x32 カラー)

### バックエンド (inta 側)

- `/api/status` エンドポイント追加
- エージェントの session-state.json やプロセス情報を読み取って返す

## UIデザイン

- 卵型フレーム（たまごっち本体風）
- 中央にドット絵キャラ1体
- フレーム下部にステータステキスト表示
- レトロだけどカラフルな雰囲気

## ポーリング

- 間隔: 5秒程度
- エンドポイント: `GET /api/status`
- レスポンス例:
  ```json
  {
    "process": "running",
    "task": "slack",
    "agentName": "Claude.Yamabe",
    "uptime": 3600
  }
  ```
